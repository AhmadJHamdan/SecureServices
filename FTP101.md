[File Transfer Protocol]([url](https://www.rfc-editor.org/rfc/rfc959.html)) (FTP) runs within the Application Layer of the TCP/IP protocol stack.

**Use Cases**<br>
- Making files accessible to a network
- Uploading files to a server
- Downloading files from a server
- Storing files on a server

**Modes**<br>
FTP has 2 modes:<br>
- Active Mode: Client initiates a Control Channel connection to destination port 21 TCP. If successful Server replies by opening a Data Channel from port 20 TCP to a port specified by the client through the PORT command.
> Server Initiates Connection to Client --> Active
- Passive Mode: Client initiates a Control Channel connection to destination port 21 TCP. The client sends a PASV command. The Server then sends its reply over the Control Channel connection. The reply contains the destination port (an ephemeral port) the client should use to establish the Data Channel.
> Only Client Initiates Connection to Server --> Passive

**Ports**<br>
- In Active Mode FTP opperates using 2 different channels (ports)<br>
  1- Control Channel (Client to Server on Destination Port 21 TCP)<br> 
  2- Data Channel (Server to Client from Source Port 20 TCP)<br>

- Similarily in Passive Mode FTP opperates using 2 different channels (ports)<br>
  1- Control Channel (Client to Server on Destination Port 21 TCP)<br>
  2- Data Channel (Client to Server on an Ephemeral Port)<br>

**Control Channel Vs Data Channel**<br>
- Control Channel is used to execute commands (login, ls ...)<br>
- Data Channel is used to transfer the actual data (files)<br>
